# Session Compaction and L2 Summary

MyClaw includes minimal L2 session compaction so long conversations do not keep injecting unbounded transcript events into prompts.

## What L2 Summary Is

`SessionCompactor` reads L2 session transcript records through `MemoryRouter`, builds a deterministic extractive summary, and writes that summary back to L2 Session Memory as a normal `MemoryRecord`:

```text
layer = session
key = session_summary:<sessionId>
title = Session summary
metadata.summaryType = l2_session_summary
```

The summary remains session-scoped. It is used for prompt recall and session continuity.

## L2 Summary vs L3 Semantic Memory

L2 summary is not long-term memory:

- It summarizes a specific session transcript.
- It is not written to Markdown semantic memory.
- It does not create facts, preferences, decisions, or project notes.
- It is not automatically promoted to L3.

L3 Semantic Memory remains the canonical place for stable facts and preferences, and writing L3 still requires explicit memory permissions.

## Prompt Behavior

`PromptAssembler` still calls `MemoryRouter.retrieveForPrompt()`, but when an L2 summary exists it prioritizes:

1. the latest L2 session summary
2. recent L2 events
3. L1/L3/L4 retrieval results according to existing policy and prompt budget

This avoids stuffing old transcript details into `[Memory Context]` while preserving recent turn-level context.

## Trigger Behavior

`QueryEngine` runs L2 compaction at the end of a turn when configured thresholds are exceeded:

```json
{
  "memory": {
    "compaction": {
      "enabled": true,
      "maxEventsBeforeCompact": 30,
      "maxCharsBeforeCompact": 20000,
      "maxSummaryChars": 4000,
      "keepRecentEvents": 8
    }
  }
}
```

The summary uses a stable `session_summary:<sessionId>` ID, so repeated compaction updates one summary record instead of creating infinite summary records.

## Deterministic Summarizer

This step does not call a real model. The summarizer is deterministic and extractive:

- preserves the latest user goal
- includes recent user messages
- summarizes tool call/result/error activity
- includes the latest assistant answer
- truncates to `maxSummaryChars`
- marks content as generated by the deterministic L2 session summarizer

## Policy and Audit

L2 summary generation is a session-internal write, so it does not require `memory.write` because it does not touch L3/L4.

Each compaction run records:

- L2 `system_event`: `session.compacted`
- AuditLog event: `session.compaction.run`

## Current Limitations

- No real LLM summarizer.
- No stable fact extraction.
- No automatic L2-to-L3 promotion.
- No embedding/vector compaction.
- No daemon or background compaction worker.

Future work can add a real summarizer, stable fact extractor, and human-reviewed promotion from L2 summaries into L3 semantic memory.
